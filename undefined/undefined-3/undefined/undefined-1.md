# 도구 관리

### 1. 도구 관리 페이지 개요

도구 관리 페이지에서는 시스템에서 사용할 수 있는 도구들을 확인하고 등록할 수 있습니다.\
![](<../../../.gitbook/assets/image (217).png>)

#### ① 상태 필터 (좌측 셀렉트 박스)

* 도구의 상태별로 목록을 필터링할 수 있습니다.
* 예: 전체 / 활성 / 비활성 / 테스트 등
* 현재 등록된 도구들의 **운영 상태 현황을 빠르게 확인**할 수 있습니다.

#### ② 도구 목록

* 등록된 도구 리스트를 확인할 수 있습니다.
* 각 도구의 이름, 버전, 상태, 카테고리 등의 정보를 제공합니다.

#### ③ 도구 추가 버튼

* 새로운 도구를 등록할 수 있습니다.
* 클릭 시 **도구 추가 화면**으로 이동합니다.

***

## &#x20;도구 추가 항목 설명

### &#x20;1. 기본 정보

<figure><img src="../../../.gitbook/assets/image (290).png" alt=""><figcaption></figcaption></figure>

#### 도구명 (시스템 식별자)

* 예: `youtube-search-tool`
* 시스템 내부에서 사용되는 **고유한 식별자**입니다.
* 중복될 수 없으며, API 및 내부 로직에서 참조됩니다.

#### 버전

* 예: `1.0.0`
* 도구의 버전을 입력합니다.
* 업데이트 및 변경 이력 관리를 위해 사용됩니다.

#### 도구 설명

* 도구의 기능과 역할을 설명합니다.
* 사용자가 도구를 선택할 때 참고하는 정보입니다.

***

### 2. 분류 정보

#### 레지스트리 타입

* 예: `NPM`
* 도구가 배포된 패키지 레지스트리를 선택합니다.
* (예: NPM, PyPI, 내부 레지스트리 등)

#### 카테고리

* 예: `웹 검색`
* 도구의 기능 분류를 선택합니다.
* 사용자가 도구를 쉽게 찾을 수 있도록 돕습니다.

***

### &#x20;3. 배포 정보

#### 배포 타입

* 예: `로컬`
* 도구가 실행되는 환경을 선택합니다.
* 예:
  * 로컬 (내부 서버 실행)
  * 클라우드
  * 컨테이너 기반

***

### &#x20;4. URL 및 이미지 정보

#### 저장소 URL

* 예: [https://github.com/](https://github.com/)...
* 도구의 소스 코드 또는 패키지 저장소 주소입니다.

#### 아이콘 URL

* 예: [https://example.com/icon.png](https://example.com/icon.png)
* 도구를 대표하는 아이콘 이미지 URL입니다.

#### 엔드포인트 URL

* 예: [http://localhost:7000/mcp/](http://localhost:7000/mcp/)
* 도구 실행을 위한 API 엔드포인트입니다.
* 시스템에서 해당 URL로 요청을 보내 도구를 호출합니다.

#### 컨테이너 이미지

* 예: `didim365/mcp-web-search:latest`
* Docker 이미지 주소를 입력합니다.
* 컨테이너 기반 배포 시 사용됩니다.
