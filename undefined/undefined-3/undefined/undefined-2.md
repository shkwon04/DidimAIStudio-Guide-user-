# 페르소나 관리

## 페르소나 관리 페이지 개요

페르소나 관리 페이지는 시스템에서  생성, 수정, 관리하는 공간입니다.

각 페르소나는 하나의 “AI 인격 설정”으로 작동하며, 모델의 응답 스타일과 목적을 결정합니다.

### ![](<../../../.gitbook/assets/image (326).png>)&#x20;

### ① 검색 기능

페르소나를 다양한 조건으로 검색할 수 있습니다.

* 이름 검색: 페르소나 제목 기준 검색
* 설명 검색: 역할/목적 설명 기반 검색
* 시스템 프롬프트 검색: 내부 AI 설정 문구 기반 검색

***

### &#x20;② 필터 기능

페르소나 목록을 조건별로 필터링합니다.

* 카테고리 (예: 의료, 법률, 개발 등)
* 공개 여부 (공개 / 비공개)
* 오름 차순 정렬

***

### ③ 페르소나 리스트

등록된 페르소나 목록을 확인할 수 있습니다.

| 항목   | 설명            |
| ---- | ------------- |
| ID   | 고유 식별자        |
| 제목   | 사용자에게 노출되는 이름 |
| 설명   | 역할 및 목적 설명    |
| 카테고리 | 분야 분류         |
| 공개   | 외부 공개 여부      |
| 생성일  | 생성된 날짜        |

<figure><img src="../../../.gitbook/assets/image (348).png" alt=""><figcaption></figcaption></figure>

#### &#x20;페르소나 제목

* 예: `Python 개발 전문가`
* 사용자에게 노출되는 공식 이름

***

#### &#x20;공개 여부

* ON: 다른 사용자도 사용 가능
* OFF: 내부 전용

***

#### &#x20;카테고리

* 예: 의료 / 법률 / 개발 / 마케팅
* 페르소나 분류 기준

***

#### ✔설명

* 해당 페르소나의 목적과 역할 요약
* 사용자 선택 시 참고 정보

***



### 시스템 프롬프트 예시&#x20;

```
<ROLE>
당신은 10년 경력의 Python 개발 전문가입니다.
</ROLE>

<INSTRUCTIONS>
사용자의 코드를 분석하고 개선점을 제안하세요.
항상 실용적이고 명확하게 답변하세요.
</INSTRUCTIONS>
```
